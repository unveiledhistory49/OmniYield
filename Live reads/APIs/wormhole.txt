### Quick Breakdown (Official from wormhole.com/docs)
- **Core TypeScript SDK + Solidity SDK + Messaging/NTT protocols**  
  100% free and open-source.  
  `npm install @wormhole-foundation/sdk` — zero cost, no API key needed.  
  Use for arbitrary cross-chain messages or Native Token Transfers (NTT).

- **Wormhole Connect (widget)**  
  Free to embed in any frontend (React, vanilla JS, etc.).  
  As few as **3 lines of code** to add a full bridging UI.  
  Powers native + wrapped token transfers across 30+ chains with one click.

- **Wormholescan API** (for querying transfers, VAAs, NTT metadata, etc.)  
  **Public & free** — no authentication, no API key, no rate-limit warnings in docs.  
  Just call `https://api.wormholescan.io/...` (full Swagger at wormholescan.io/#/developers/api-doc).

- **Wormhole Queries** (pull on-chain data cross-chain without gas)  
  Currently in closed beta → request a free API key via their form (approved quickly).  
  No pricing, costs, or quotas mentioned anywhere in docs.  
  Use with `X-API-Key` header on the Query Proxy.

### Can You Use It for Cross-Chain Routing / Bridging?
**Yes — this is one of Wormhole’s strongest use cases.**

| Feature                  | What it does                                                                 | Routing Style                  | Best For                  |
|--------------------------|-----------------------------------------------------------------------------|--------------------------------|---------------------------|
| **Native Token Transfers (NTT)** | Lock/mint or burn/mint native tokens across chains (no wrapping needed on many paths) | Direct, guaranteed paths      | Simple, high-volume bridges |
| **Wormhole Connect widget** | Drop-in UI for users to bridge any supported asset                           | User-facing, multi-chain      | Frontends & wallets       |
| **Settlement (Intent-based)** | Users submit intent → solvers compete to find best route/price/speed (auctions) | Smart routing (like aggregators) | Optimal execution, gasless options |
| **Generic Messaging**    | Arbitrary data + calls (bridge + execute contract on destination)            | Custom logic                  | Advanced dApps / intents  |

- **30–40+ chains** (Ethereum, Solana, Base, Arbitrum, BNB, Polygon, Sui, Aptos, Cosmos chains, etc.).
- **Speed**: Often seconds with automatic relayers (you can run your own or use theirs — still free at protocol level).
- **Only real costs**: Source-chain gas + tiny message fee (often ~$0.01–$0.10). Destination gas can be sponsored/relayed for “gasless” UX.

### Getting Started (Zero Cost)
1. Docs hub: https://wormhole.com/docs/
2. Quick widget: https://wormhole.com/connect (or npm `@wormhole-foundation/wormhole-connect`)
3. TypeScript SDK tutorial: https://wormhole.com/docs/tools/typescript-sdk/get-started/
4. Full bridging example repo: GitHub wormhole-foundation (demo-cross-chain-token-transfer, etc.)
5. For Queries beta key: fill the form linked in their get-started guide.

Wormhole is the **foundational messaging layer** that powers many other bridges and aggregators (including parts of Portal Bridge at portalbridge.com). It’s broader than deBridge (full messaging + data + queries) but equally free to start with and production-ready.
